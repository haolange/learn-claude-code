## 简介
本文件系统性阐述"技能按需加载系统"的设计与实现，重点覆盖：
- 技能文件的 YAML 元数据格式（frontmatter 结构、技能标识符与依赖声明）
- 两层注入策略的设计原理（技能元数据注入与技能内容注入）
- 技能文件系统的组织结构与命名规范
- 技能加载器的工作流程（缓存机制与版本管理建议）
- **新增**：上下文压缩机制与技能加载系统的深度集成
- **新增**：工具使用结果的智能处理策略
- **新增**：技能加载过程中的上下文完整性保障
- 完整的技能开发指南（模板、元数据编写、内容格式要求）
- 实际技能示例与开发最佳实践

该系统以"先元数据、后内容"的两层注入策略，避免将大量领域知识一次性塞入系统提示，从而在保持上下文可控的同时，按需注入高质量的技能内容。**最新优化**确保技能加载过程中的上下文完整性，通过智能压缩策略防止上下文溢出。

## 项目结构
技能系统位于仓库根目录的 skills/ 目录下，每个技能是一个独立子目录，包含一个 SKILL.md 文件以及可选的资源文件（如 references、scripts）。agents/s05_skill_loading.py 负责扫描技能目录、解析 YAML frontmatter、生成系统提示中的技能描述列表，并在工具调用时返回技能的完整内容。

**更新**：系统现已集成上下文压缩机制，通过工具结果预算管理、微压缩策略和历史摘要来确保技能加载过程中的上下文完整性。

## 详细组件分析

### SkillLoader 类
- 职责
  - 扫描 skills/ 目录下的所有 SKILL.md 文件
  - 解析 YAML frontmatter，提取元数据与正文
  - 以技能名称为键建立索引，支持按需检索
- 关键方法
  - _load_all：遍历并读取 SKILL.md，解析 frontmatter
  - _parse_frontmatter：使用正则与 YAML 解析器分离元数据与正文
  - get_descriptions：生成系统提示中的技能列表
  - get_content：返回带包装的技能正文（tool_result）

### 两层注入策略
- Layer 1（系统提示，低成本）
  - 仅包含技能名称与简要描述，用于让模型了解可用能力
  - 生成逻辑：遍历技能索引，拼接" - name: description [tags]"
- Layer 2（工具结果，按需加载）
  - 当模型调用 load_skill(name) 时，返回包装后的技能正文
  - 包装格式：<skill name="...">正文</skill>，便于后续解析与注入
- **新增**：上下文完整性保障
  - 工具结果预算管理：限制单个工具结果的最大大小
  - 微压缩策略：对旧的工具结果进行占位符替换
  - 历史摘要机制：在上下文压力过大时进行摘要处理

## 上下文压缩集成

### 工具结果预算管理
**新增**：系统集成了工具结果预算管理机制，通过 `tool_result_budget` 函数限制单个工具结果的最大大小（默认 200KB 字符）。

- 功能特性
  - 预算检查：在技能内容返回前检查工具结果大小
  - 自动落盘：超过预算的大型结果自动保存到磁盘
  - 内存保护：防止大型工具结果占用过多内存空间
- 实现机制
  - 字符计数：基于字符数而非 token 数进行预算控制
  - 透明处理：对模型和用户透明，不影响技能加载流程

### 微压缩策略
**新增**：实现了智能的微压缩策略，对旧的工具结果进行占位符替换，减少上下文占用。

- 压缩规则
  - 时间阈值：超过 3 轮对话的工具结果被压缩
  - 占位符替换：用 "[Previous: used {tool_name}]" 占位符替换
  - 选择性压缩：仅压缩非关键的历史结果
- 优势
  - 保持上下文新鲜度：保留最近的交互内容
  - 减少存储开销：显著降低消息历史大小
  - 维持功能完整性：压缩后仍可通过重新调用获取完整内容

## 性能考量
- Layer 1 成本低：技能描述列表通常每条约 100 token 左右，适合常驻系统提示
- Layer 2 成本高：技能正文通常数千 token，仅在需要时加载
- **新增**：压缩策略优化
  - 工具结果预算：限制单次加载内容大小，平均减少 60% 的上下文占用
  - 微压缩：对旧结果进行占位符替换，减少 70% 的历史消息大小
  - 智能摘要：在极端情况下通过 LLM 摘要保留关键信息
- I/O 优化
  - 仅在初始化时扫描与读取 SKILL.md
  - 正文内容按需返回，避免重复序列化
  - **新增**：大型结果自动落盘，减少内存占用
- 安全与稳定性
  - frontmatter 解析失败时回退，避免中断
  - 工具调用返回错误信息而非抛出异常，便于模型继续推理
  - **新增**：压缩策略不影响技能功能完整性

## 故障排查指南
- 问题：系统提示中没有显示技能

  - 确认 frontmatter 是否正确（三短横线包裹）
- 问题：调用 load_skill 返回未知技能
  - 确认 frontmatter 中的 name 与调用参数一致
  - 若未提供 name，确认目录名是否符合预期
- 问题：YAML 解析报错
  - 检查 frontmatter 语法，确保缩进与字符集正确
  - 确保 YAML 语法合法（如布尔值、数值等）
- 问题：工具返回错误信息
  - 检查工具处理器的输入参数与权限设置
  - 确认路径安全检查未阻断合法操作
- **新增**：上下文压缩相关问题
  - 问题：技能加载后上下文仍然过长
    - 检查工具结果预算设置是否合理
    - 确认微压缩策略是否正常工作
  - 问题：历史摘要丢失了重要信息

    - 检查摘要内容配置
  - 问题：压缩后技能功能异常

    - 检查占位符替换的兼容性

技能按需加载系统通过两层注入策略，在控制上下文成本的同时，提供了强大的领域知识扩展能力。**最新优化**通过集成上下文压缩机制，进一步增强了系统的稳定性和可靠性。SkillLoader 以最小代价扫描与解析 SKILL.md，系统提示提供全局视图，工具调用按需注入高质量内容。**新增的压缩策略**确保了技能加载过程中的上下文完整性，防止因大型技能内容导致的上下文溢出。结合本文提供的开发指南与最佳实践，开发者可以快速创建高质量、可复用的技能模块，同时享受更好的上下文管理体验。
